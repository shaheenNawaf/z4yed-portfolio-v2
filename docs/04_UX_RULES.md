# UX Rules

Version: 1.0

---

# Philosophy

Reduce thinking.

Reduce reading.

Reduce friction.

Increase confidence.

---

# Rules

Every section answers one recruiter question.

One primary CTA per screen.

Progressive disclosure over information overload.

Visuals before paragraphs.

Results before responsibilities.

Projects before experience.

Evidence before claims.

---

# Reading Rules

Maximum paragraph

4 lines.

Prefer

Cards

Lists

Metrics

Screenshots

Expandable sections.

---

# Interaction Rules

Animations guide attention.

Never distract.

Hover should reveal.

Click expands.

Scrolling should feel natural.

---

# Navigation

Navbar hidden until hero exits.

Sticky after reveal.

Minimal links only.

---

# Mobile

Collapse aggressively.

Prioritize scrolling over tapping.

Never horizontal scroll.

---

# Anti-Friction

No unnecessary clicks.

No hidden navigation.

No walls of text.

No decorative content.

---

# Acceptance Criteria

□ Easy to scan

□ Minimal reading

□ Low cognitive load

□ Mobile friendly

A reviewer should be able to answer YES to every question below.

□ Is the purpose of this document immediately clear?

□ Does this document remove ambiguity?

□ Can an AI follow this without making assumptions?

□ Does this align with the portfolio's positioning?

□ Does this prioritize recruiter conversion?

□ Does this avoid unnecessary complexity?

□ Is every decision justified?
