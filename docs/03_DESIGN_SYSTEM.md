# Design System

Version: 1.0
Status: Draft

---

# Purpose

Define a consistent visual language for the entire portfolio.

Consistency is more important than creativity.

---

# Design Philosophy

Premium.

Minimal.

Modern.

Editorial.

Product-first.

The portfolio should resemble companies like Linear, Vercel, Stripe and Raycast.

Never resemble a template portfolio.

---

# Typography

Font Family

Satoshi (self-hosted via @font-face from Fontshare)

Weights

400 Regular (body)
500 Medium (subheadings, emphasis)
700 Bold (headings, display)

Fallback

system-ui

Maximum Fonts

ONE

Do not introduce secondary fonts.

Exception

System monospace stack is permitted for code/terminal UI elements (tags, dates, labels, watermark).
This is an intentional design exception — not a secondary brand font.

# Colors

Primary Background

Near Black

Primary Text

White

Secondary Text

Muted Gray

Accent

Orange

Secondary Accent

Blue

Semantic colors only.

Never introduce additional accent colors.

---

# Layout

Content Width

Consistent max-width.

Large breathing room.

8pt spacing system.

Responsive first.

---

# Components

Rounded corners

Medium radius.

Soft shadows.

Minimal borders.

Large screenshots.

Minimal icon usage.

---

# Buttons

Primary

Solid.

Secondary

Ghost.

Never more than two CTAs together.

---

# Cards

Large visuals.

Minimal text.

Hover elevation.

Clickable.

---

# Icons

Outline style only.

Consistent size.

No emoji.

---

# Accessibility

WCAG AA contrast.

Keyboard navigation.

Visible focus states.

Semantic HTML.

---

# Acceptance Criteria

□ One font only

□ Consistent spacing

□ Consistent colors

□ Premium aesthetic

□ Mobile responsive

# Acceptance Criteria

A reviewer should be able to answer YES to every question below.

□ Is the purpose of this document immediately clear?

□ Does this document remove ambiguity?

□ Can an AI follow this without making assumptions?

□ Does this align with the portfolio's positioning?

□ Does this prioritize recruiter conversion?

□ Does this avoid unnecessary complexity?

□ Is every decision justified?
